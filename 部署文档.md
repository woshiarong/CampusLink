## 📥 环境安装

### 1. JDK 安装

#### Windows

1. **下载地址**：
   - Oracle JDK: https://www.oracle.com/java/technologies/downloads/
   - OpenJDK: https://adoptium.net/zh-CN/temurin/releases/
   - 推荐：OpenJDK 11 或 17 LTS

2. **安装步骤**：
   - 下载对应系统的安装包（Windows x64）
   - 运行安装程序，按提示完成安装
   - 记住安装路径（如：`C:\Program Files\Java\jdk-11`）

3. **配置环境变量**：
   - 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"
   - 新建系统变量：
     - 变量名：`JAVA_HOME`
     - 变量值：JDK 安装路径（如：`C:\Program Files\Java\jdk-11`）
   - 编辑 `Path` 变量，添加：`%JAVA_HOME%\bin`
   - 验证：打开命令行，输入 `java -version`

#### macOS

```bash
# 使用 Homebrew 安装
brew install openjdk@11

# 配置环境变量（添加到 ~/.zshrc 或 ~/.bash_profile）
export JAVA_HOME=$(/usr/libexec/java_home -v 11)
export PATH=$JAVA_HOME/bin:$PATH
```

#### Linux

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-11-jdk

# CentOS/RHEL
sudo yum install java-11-openjdk-devel

# 验证
java -version
```

### 2. Maven 安装

#### Windows

1. **下载地址**：
   - https://maven.apache.org/download.cgi
   - 下载 `apache-maven-3.9.x-bin.zip`

2. **安装步骤**：
   - 解压到目录（如：`C:\Program Files\Apache\maven`）
   - 配置环境变量：
     - 新建系统变量：
       - 变量名：`MAVEN_HOME`
       - 变量值：Maven 解压路径（如：`C:\Program Files\Apache\maven`）
     - 编辑 `Path` 变量，添加：`%MAVEN_HOME%\bin`
   - 验证：打开命令行，输入 `mvn -version`

#### macOS

```bash
# 使用 Homebrew 安装
brew install maven

# 验证
mvn -version
```

#### Linux

```bash
# Ubuntu/Debian
sudo apt install maven

# CentOS/RHEL
sudo yum install maven

# 验证
mvn -version
```

2. **安装步骤**：

```bash
# CentOS/RHEL

# 验证
node -v
npm -v
```

### 4. MySQL 安装

#### Windows

1. **下载地址**：
   - 官网：https://dev.mysql.com/downloads/mysql/
   - 选择 MySQL Community Server 8.0.x

2. **安装步骤**：
   - 下载 MySQL Installer for Windows
   - 运行安装程序，选择 "Developer Default" 或 "Server only"
   - 设置 root 用户密码（记住此密码，后续配置需要）
   - 完成安装

3. **验证**：
   - 打开命令行，输入：`mysql --version`
   - 如果提示找不到命令，需要将 MySQL 的 bin 目录添加到 PATH
   - 默认路径：`C:\Program Files\MySQL\MySQL Server 8.0\bin`

#### macOS

```bash
# 使用 Homebrew 安装
brew install mysql@8.0

# 启动服务
brew services start mysql@8.0

# 设置 root 密码
mysql_secure_installation
```

#### Linux

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server

# 启动服务
sudo systemctl start mysql
sudo systemctl enable mysql

# 设置 root 密码
sudo mysql_secure_installation
```

## ⚙️ 项目配置

### 1. 数据库配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/xiaoyuan_wenchuang?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root          # 修改为你的 MySQL 用户名
    password: root          # 修改为你的 MySQL 密码
```

### 2. 文件上传配置

文件上传目录已配置为相对路径（`./uploads`），会在项目根目录自动创建 `uploads` 文件夹，无需修改。

### 3. 前端代理配置

前端已配置代理到后端，无需修改。如需修改后端地址，编辑 `frontend/vite.config.js`：

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',  // 修改为你的后端地址
      changeOrigin: true
    }
  }
}
```

## 🚀 运行步骤

### 方式一：使用批处理脚本（Windows 推荐）

#### 1. 初始化数据库

双击运行 `初始化数据库.bat`

- 脚本会自动检查 MySQL 是否可用
- 提示输入 MySQL root 密码
- 自动创建数据库并导入初始数据

#### 2. 启动后端

双击运行 `启动后端.bat`

- 脚本会自动检查 Java 和 Maven 环境
- 自动编译并启动 Spring Boot 应用
- 后端运行在：`http://localhost:8080/api`

#### 3. 启动前端

双击运行 `启动前端.bat`

- 脚本会自动检查 Node.js 和 npm 环境
- 首次运行会自动安装依赖（`npm install`）
- 启动开发服务器
- 前端运行在：`http://localhost:3000`

---

### 方式二：命令行运行

#### 1. 初始化数据库

```bash
# Windows
mysql -u root -p < database\init.sql

# macOS/Linux
mysql -u root -p < database/init.sql
```

输入 MySQL root 密码，等待导入完成。

#### 2. 启动后端

```bash
# 进入后端目录
cd backend

# 启动 Spring Boot（首次运行会自动下载依赖）
mvn spring-boot:run

# 或者先编译再运行
mvn clean package
java -jar target/xiaoyuan-wenchuang-1.0.0.jar
```

后端启动后，访问：`http://localhost:8080/api`

#### 3. 启动前端

打开新的命令行窗口：

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

前端启动后，访问：`http://localhost:3000`

## ❓ 常见问题

### 1. 数据库连接失败

**问题**：后端启动时报错 "Communications link failure"

**解决方案**：
- 检查 MySQL 服务是否启动
  - Windows：服务管理器查看 MySQL 服务状态
  - macOS：`brew services list`
  - Linux：`sudo systemctl status mysql`
- 检查 `application.yml` 中的数据库配置是否正确
- 检查 MySQL 用户名和密码是否正确
- 检查端口 3306 是否被占用

### 2. 端口被占用

**问题**：启动时报错 "Port 8080 is already in use" 或 "Port 3000 is already in use"

**解决方案**：
- **后端端口**：修改 `backend/src/main/resources/application.yml` 中的 `server.port`
- **前端端口**：修改 `frontend/vite.config.js` 中的 `server.port`
- 或者关闭占用端口的程序

### 3. Maven 下载依赖慢

**解决方案**：
- 配置国内镜像源，编辑 `~/.m2/settings.xml`（如不存在则创建）：
  ```xml
  <settings>
    <mirrors>
      <mirror>
        <id>aliyun</id>
        <mirrorOf>central</mirrorOf>
        <url>https://maven.aliyun.com/repository/public</url>
      </mirror>
    </mirrors>
  </settings>
  ```

### 4. npm 安装依赖慢

**解决方案**：
```bash
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 或使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### 5. 前端页面空白或报错

**解决方案**：
- 检查后端是否正常启动
- 检查浏览器控制台错误信息
- 清除浏览器缓存
- 检查 `frontend/vite.config.js` 中的代理配置

### 6. 文件上传失败

**解决方案**：
- 检查项目根目录是否有 `uploads` 文件夹（会自动创建）
- 检查文件夹权限（Linux/macOS 需要写权限）
- 检查磁盘空间是否充足

### 7. 登录失败

**解决方案**：
- 确认使用正确的账号密码（见[默认账号](#默认账号)）
- 检查后端日志是否有错误信息
- 确认数据库已正确初始化
- 清除浏览器 localStorage 后重试

### 8. 批处理脚本闪退

**解决方案**：
- 检查环境变量是否正确配置
- 在脚本末尾添加 `pause` 命令查看错误信息
- 手动在命令行运行脚本查看详细错误

### 9. 跨域问题

**问题**：前端请求后端接口时出现 CORS 错误

**解决方案**：
- 后端已配置跨域支持，检查 `WebConfig.java` 中的配置
- 确认前端访问的是正确的后端地址

### 10. 数据库字符集问题

**问题**：中文显示乱码

**解决方案**：
- 确认数据库和表使用 `utf8mb4` 字符集
- 检查 `application.yml` 中的连接字符串包含 `useUnicode=true&characterEncoding=utf8`
- 重新执行 `database/init.sql` 初始化脚本

## 📝 注意事项

1. **首次运行**：
   - 必须先初始化数据库
   - 后端启动需要一些时间（下载依赖）
   - 前端首次运行需要安装依赖（`npm install`）

2. **开发模式**：
   - 后端修改代码后需要重启
   - 前端修改代码会自动热更新

3. **生产部署**：
   - 修改数据库密码
   - 修改 JWT 密钥（`application.yml` 中的 `jwt.secret`）
   - 前端打包：`npm run build`，将 `dist` 目录部署到服务器

4. **文件上传**：
   - 上传的文件保存在项目根目录的 `uploads` 文件夹
   - 生产环境建议使用对象存储服务（OSS）
